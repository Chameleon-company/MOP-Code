Scenario

As a wedding planner, I want to identify suitable wedding venues in Melbourne, understand the necessary permits, and locate photogenic areas for wedding photos, so I can provide personalized recommendations to clients and ensure a smooth planning process.

What this use case will teach you

At the end of this use case you will:

  • Data wrangling and preprocessing
  • Geospatial analysis and mapping
  • Data visualization
  • Wedding industry knowledge
Introduction or background relating to problem

Melbourne is a popular destination for weddings, with numerous venues offering unique settings and backdrops. However, finding the perfect venue can be overwhelming, especially when considering factors like permits, photography opportunities, and previous wedding events. This use case aims to leverage data science techniques to streamline the wedding planning process, providing valuable insights for wedding planners and clients alike.

Background To solve this problem, we will utilize the following datasets:

  • Event Permits (2014-2018): This dataset contains information on event permits issued by the City of Melbourne, including weddings, film shoots, and photo shoots. (https://data.melbourne.vic.gov.au/explore/dataset/event-permits-2014-2018-including-film-shoots-photo-shoots-weddings-christmas-pa/information/)
  • Outdoor Artworks: This dataset features information on outdoor artworks in Melbourne, which can serve as photogenic backdrops for wedding photos. (https://data.melbourne.vic.gov.au/explore/dataset/outdoor-artworks/information/

)

  • Public Artworks, Fountains, and Monuments: This dataset provides information on public artworks, fountains, and monuments in Melbourne, offering additional photography opportunities. (https://data.melbourne.vic.gov.au/explore/dataset/public-artworks-fountains-and-monuments/information/)

By combining these datasets, we can identify popular wedding venues, understand permit requirements, and pinpoint photogenic areas for wedding photos, ultimately enhancing the wedding planning experience.

# Function to download and convert to DataFrame
def download_data(link):
    response = requests.get(link)
    response.raise_for_status()
    data = response.json()
    records = data['results']
    return pd.DataFrame(records)

2.Clean and Process the data

Cleaning and preparing the data for analysis by normalizing text, handling missing values, and removing duplicates. First, I created a normalize_text function that strips leading/trailing whitespace and converts text to lowercase in specified columns to maintain consistency. Then, I convert dictionary-like columns (e.g., geo_point_2d, geo_shape, and co_ordinates) to string format so they can be more easily processed. After that, I handle missing values by filling them with the string 'unknown' to avoid errors during analysis. Finally, I remove any duplicate entries to ensure the data is clean and unique. After cleaning the data, I print the first few rows of each dataset to verify the changes.

# Normalize text (e.g., strip leading/trailing whitespace, convert to lowercase)
def normalize_text(df, columns):
    for col in columns:
        df[col] = df[col].str.strip().str.lower()
    return df

df_permits = normalize_text(df_permits, ['title', 'location'])
df_artworks = normalize_text(df_artworks, ['title', 'location'])
df_public_artworks = normalize_text(df_public_artworks, ['name', 'address_point'])

# Convert dict columns to string
df_artworks['geo_point_2d'] = df_artworks['geo_point_2d'].astype(str)
df_artworks['geo_shape'] = df_artworks['geo_shape'].astype(str)
df_public_artworks['co_ordinates'] = df_public_artworks['co_ordinates'].astype(str)

# Handle missing values
df_permits.fillna('unknown', inplace=True)
df_artworks.fillna('unknown', inplace=True)
df_public_artworks.fillna('unknown', inplace=True)

# Remove duplicates
df_permits.drop_duplicates(inplace=True)
df_artworks.drop_duplicates(inplace=True)
df_public_artworks.drop_duplicates(inplace=True)

Conclusion:

The wedding planner use case demonstrates how data science techniques, such as geospatial analysis and data visualization, can significantly enhance the planning process. By leveraging publicly available datasets, wedding planners can streamline venue selection, ensure compliance with permit requirements, and offer personalized recommendations for photogenic locations. The integration of data-driven insights allows planners to provide a more efficient and tailored service to clients.

Recommendations:

  1. Expand Data Sources: Consider incorporating additional datasets, such as traffic patterns, weather forecasts, and customer reviews, to offer a more comprehensive planning experience.
  2. Develop User-Friendly Tools: Create interactive tools that allow planners to visualize and explore venue options dynamically. This could include features like real-time availability, cost comparison, and permit application integration.
  3. Collaboration with Local Authorities: Partner with local government agencies to keep datasets updated and offer wedding planners access to the latest permit and venue information.
  4. Enhance Photogenic Location Identification: Use image recognition algorithms to automatically detect photogenic spots based on aesthetic qualities, which could improve the accuracy of location recommendations.

Limitations:

  1. Data Availability: The analysis relies heavily on public datasets, which may not always be updated or comprehensive enough to cover all relevant venues and permit information.
  2. Subjective Preferences: While data can suggest popular venues and photogenic spots, personal preferences such as the style of photography and individual venue tastes may not always align with the recommendations.
  3. Permit Process Variability: Permit requirements may vary based on the specific event or location, and relying solely on historical data may not capture real-time changes in rules or application processes.

Next Steps:

  1. Real-Time Data Integration: Incorporate real-time data feeds from venue providers and local authorities to ensure planners have access to the most current information.
  2. Feedback System: Develop a client feedback mechanism to capture user preferences and experiences, which could be fed into the model to improve future recommendations.
  3. Mobile Application Development: Build a mobile app that offers wedding planners and clients access to this data on the go, complete with mapping features and notifications for permit deadlines.
  4. Machine Learning Implementation: Use machine learning models to predict the popularity of new venues or upcoming events based on past data, providing planners with cutting-edge insights.
